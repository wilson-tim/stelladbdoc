Purpose
-------
Common procedures and functions for First Choice (e.g. error processing, generic
insert) used at First Choice.

Notes
-----
1.When called on its own will run with invoker's privileges. This allows dynamic
'DDL' and 'DML' to be done in various different schemas.
2.If a procedure is invoked from another package that has 'definer' rights then
the procedure in this package will also be run with the definer's rights not
the invoker's rights.

    p_out_insert_    Count of rows inserted.                                       OUT
    name

    Notes:
    1.Only copies columns that MATCH on both tables.
    2.Does not do a COMMIT. That is up to the calling environment.
    3.if 'WHERE' clause passed then it must NOT include the 'WHERE' at the beginning.
    4.If owner not passed then only the table name is used, i.e. assume in same schema
      connected to or taken care of by synonyms.
    5.If a large rollback is to be used, then a 'COMMIT' or 'ROLLBACK' must be issued
      immediately before the execution of this procedure as the setting of a rollback
      segment must be the first statement in a transaction.
     6.Target table must have a primary key constraint to ensure only missing records are
       inserted. Also, primary key columns must match on both target and source tables.

    Example Call
